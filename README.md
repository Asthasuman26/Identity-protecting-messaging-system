# **Identity Protecting Messaging System**

A decentralized, end-to-end encrypted messaging platform built for true anonymity and privacy.

In a world where data is constantly tracked, analyzed, and permanently stored, the Identity Protecting Messaging System redefines what secure communication should look like.

This platform is designed to provide fully anonymous group communication, where both your messages and your identity remain protected — even from the developers of the platform.

## *Overview*

The Identity Protecting Messaging System enables users to communicate securely in temporary, private group chats without:

Creating accounts

Sharing personal information

Using usernames

Leaving stored chat history

Being tracked or profiled

Every session is anonymous. Every message is encrypted. Every identity is temporary.

## *Core Principles*

This project is built on five core principles:

1. Anonymity by Default

Users never create accounts or provide personal data.

2. End-to-End Encryption

Messages are encrypted before transmission and can only be read by intended participants.

3. Multi-Layer Identity Protection

Randomized, session-based identities prevent traceability.

4. Decentralized (Hybrid) Infrastructure

Communication is powered through decentralized messaging transport while maintaining performance and reliability.

5. Zero Data Retention

No stored messages. No metadata storage. No tracking systems.

## *How the Application Works*
Private Room Creation

A user creates a chat room instantly.

The system generates a unique 6-digit room code.

This room code acts as the only access key to the chat.

Anonymous Member Access

Users join using the room code.

Upon joining, each member receives a randomly generated 4-digit identity.

This identity is:

Unique per room

Unique per session

Non-persistent

When the session ends, the identity disappears.

Secure Message Transmission

Messages are transmitted through the XMTP network using full end-to-end encryption.

On top of that, the platform implements a custom-built encryption layer, specifically designed to:

Obfuscate user identities

Protect system-level data

Strengthen anonymity

Add an additional cryptographic security barrier

Even the developers of this system cannot access user messages or identities.

## *Integrated Crypto Payments*

The platform includes a crypto payment system that allows users to:

This enables seamless, privacy-preserving financial interaction alongside encrypted communication.
